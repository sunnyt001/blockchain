01
TỔNG QUAN HỆ THỐNG
IDEX là một ứng dụng mô phỏng sàn giao dịch blockchain tích hợp đầy đủ trong một file HTML duy nhất. Hệ thống gồm 2 vai trò chính: Exchange User (mua/bán) và Miner (đào block xác nhận giao dịch), kết nối với nhau qua một SharedState dùng chung.
// LUỒNG DỮ LIỆU CHÍNH
* Tất cả state được chia sẻ qua object SharedState — bridge trung tâm giữa Exchange và Miner
ℹ️
Single-file architecture: Toàn bộ CSS, HTML, JavaScript (~2300 dòng) được đóng gói trong một file duy nhất. Không cần server, không cần build tool — mở trực tiếp trên trình duyệt.
02
DID AUTHENTICATION
Thay vì username/password truyền thống, IDEX dùng Decentralized Identifier (DID) theo chuẩn W3C. Người dùng chứng minh quyền sở hữu danh tính bằng cách ký một challenge ngẫu nhiên từ server bằng private key của mình.
// 4 BƯỚC XÁC THỰC
STEP 01
Nhập DID — Validate format did:*:*
↓
STEP 02
Blockchain lookup — Lấy DID Document + Public Key
↓
STEP 03
Sign Challenge — Ký chuỗi ngẫu nhiên bằng Private Key
↓
STEP 04
Verify — Xác minh chữ ký với Public Key trên chain

Bảo mật challenge: Challenge được tạo ngẫu nhiên mỗi phiên → không thể replay lại chữ ký cũ để giả mạo.

05
MEMPOOL
Mempool (Memory Pool) là vùng chứa tạm thời cho các giao dịch chờ được đào. Trong IDEX, SharedState.mempool là mảng trung tâm được cả Exchange lẫn Miner truy cập.
Đặc điểm Mempool
● Tự động refill ngẫu nhiên mỗi 5 giây (nếu < 12 TX)
● TX có flag selected → được đào vào block kế tiếp
● TX có flag fromExchange → đánh dấu nguồn gốc
● Sau khi đào xong → TX được xóa khỏi mempool tự động
// SHARED STATE OBJECT
const SharedState = {
  mempool: [],       // Tất cả TX đang chờ
  blockchain: [],    // Các block đã xác nhận
  balances: {
    btc: 0.842,
    usdt: 12450.00
  },
  // Reactive: notify listeners
  addTx(tx) { ... },
  removeTxs(ids) { ... },
  addBlock(block) { ... }
};
06
MERKLE TREE
Merkle Tree là cấu trúc dữ liệu dạng cây nhị phân hash. Mỗi lá là hash SHA-256 của một TX. Các node cha là hash của 2 node con ghép lại. Root cuối cùng (Merkle Root) được lưu vào block header, đại diện cho toàn bộ tập TX trong block.

07
MINING ENGINE — PROOF OF WORK
Mining là quá trình tìm giá trị nonce sao cho SHA-256 hash của block header bắt đầu bằng đúng 4 số 0 (difficulty = 4). Mỗi lần thử là một lần hash — có thể cần hàng trăm nghìn lần thử.

08
BLOCKCHAIN
Blockchain là danh sách các block được liên kết với nhau bằng hash. Mỗi block chứa hash của block trước (previousHash), tạo thành chuỗi bất biến. Sửa bất kỳ block nào sẽ làm vỡ toàn bộ chuỗi sau đó.

BẢO MẬT ĐA LỚP
IDEX mô phỏng 5 lớp bảo mật độc lập, bổ sung cho nhau. Mỗi lớp bảo vệ một khía cạnh khác nhau của hệ thống — từ xác thực danh tính đến tính toàn vẹn của chuỗi block.
